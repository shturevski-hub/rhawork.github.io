Workbook Content Administration Guidelines

This resource provides instructions for authorized personnel to maintain and update the internal Workbook content. **All content is managed by directly editing the corresponding HTML files.**

‚ùó Important Policy on Updates

Content integrity is paramount. All changes must be reviewed and documented prior to implementation. Always work on a copy of the file before applying changes to the live version.

Verify that changes are displayed correctly by refreshing the browser after saving any file.

1. Managing Visit Types and Scheduling Data

To modify appointment names, durations, or descriptions:

  1. Open the file: visitTypes.html in a text editor.
  2. Locate the table body, typically marked with <tbody id="visit-types-table-body">.
  3. **To Add/Delete:** Add or remove the full <tr>...</tr> row structure.
  4. **To Edit:** Modify the content within the <td> tags of an existing row.
  5. Save the visitTypes.html file.

2. Managing Self-Pay Prices

To update prices, services, or lab codes (including Base Rates, Add-ons, and Vaccines):

  1. Open the file: selfPayPrices.html in a text editor.
  2. Locate the appropriate table section (e.g., id="base-rate-table-body", id="addon-table-body").
  3. Prices are hardcoded directly into the HTML. Edit the content of the relevant <td class="price-column"> tags for price changes.
  4. Save the selfPayPrices.html file.

3. Managing Internal Guidelines and Notes

To update Provider Notes, Internal Guidelines, or resource lists:

  1. Open the file: additionalInfo.html in a text editor.
  2. Locate the relevant content within the <main> section (organized by <section class="info-card">).
  3. Directly edit the text content or adjust the structure of the lists (<ul>, <li>).
  4. Save the additionalInfo.html file.

4. Core Site Elements (Navigation, Branding)

Changes to site title, navigation links, or the main welcome message (Home):

  1. Open index.html (for Home page content) or all HTML files (for navigation changes).
  2. For Navigation: Edit the list of <a href="..."> links inside the <nav> tag in the header.
  3. For Branding/Style: Changes to the site's colors, fonts, or overall layout must be made in the style.css file.
  4. Save the updated file(s).
